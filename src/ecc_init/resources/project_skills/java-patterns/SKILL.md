# Java 项目规范

- 沿用项目 Java 版本、构建工具和代码风格，不引入不兼容语法。
- 类和方法职责清楚，优先构造器注入，避免隐藏的全局可变状态。
- 使用明确领域类型和异常；不要用 `null` 或通用异常掩盖业务状态。
- 集合返回值和并发访问语义明确，避免暴露可变内部集合。
- 资源使用 try-with-resources；线程、连接池和事务边界清晰。
- 日志不记录密码、令牌和敏感个人数据。
- 新功能补充 JUnit/项目既有测试，覆盖边界、异常和并发敏感路径。
