# Python 项目规范

- 先读取 `pyproject.toml`、依赖文件和现有代码风格。
- 公共接口和复杂函数补充清晰类型标注，避免无必要的 `Any`。
- 使用明确异常类型，不用空 `except`，不静默吞错。
- 文件、数据库连接、锁和网络资源使用上下文管理或可靠清理。
- 异步路径避免阻塞 I/O；同步和异步接口不要混用得难以推断。
- 配置与密钥从项目既有配置系统读取，不硬编码。
- 新功能使用 pytest 或项目既有框架补充正常、边界和失败路径测试。
- 只在复杂业务规则、兼容处理和非直观实现处写中文注释。
