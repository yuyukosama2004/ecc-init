# FastAPI 项目规范

- 路由层处理 HTTP 输入输出，业务逻辑放在 service/use-case 层，数据访问沿用项目现有结构。
- 使用 Pydantic 模型校验请求和响应；不要直接信任字典或外部 API 数据。
- 认证和授权分开检查，资源归属校验不能只依赖“已登录”。
- 数据库会话生命周期明确；涉及多步写入时使用事务并处理回滚。
- 异步路由中不要调用阻塞 I/O；外部请求设置超时和错误映射。
- 对外错误信息简洁，详细异常只进入安全日志，不泄漏密钥或内部结构。
- 新接口至少覆盖：成功、参数错误、未认证/无权限、业务冲突和依赖失败。
- 修改 CORS、认证配置、数据库迁移或依赖前必须向用户确认。
