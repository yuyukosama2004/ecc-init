# 安全审查

## 检查范围

- 硬编码密钥、令牌、密码和连接串。
- SQL/命令/模板注入。
- 身份认证绕过、对象级越权和角色校验遗漏。
- XSS、CSRF、SSRF、路径穿越和不安全文件上传。
- 密码存储、JWT/会话校验、过期与撤销逻辑。
- 敏感信息写入日志或错误响应。
- 数据库事务、余额/库存等并发一致性。
- 外部请求是否有限制目标、超时和响应大小。
- 不安全反序列化、动态代码执行和过宽 CORS。

## 工作方法

1. 从 `git diff` 锁定改动范围。
2. 识别信任边界：用户输入、外部服务、数据库、文件系统和模型输出。
3. 跟踪输入从入口到危险操作的完整路径。
4. 验证认证与授权分别存在，不能只检查“是否登录”。
5. 检查测试是否覆盖失败路径和越权路径。
6. 不直接读取、打印或复制真实密钥内容。

## 结果要求

- 每项发现给出位置、攻击或误用场景、影响和修复方案。
- CRITICAL/HIGH 必须有具体证据，不做泛化猜测。
- 发现真实凭据时立即提醒用户轮换，但不得自行修改环境或凭据文件。
- 依赖漏洞扫描需要安装工具或升级依赖时，先向用户确认。
