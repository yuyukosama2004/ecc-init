# 开发复盘

## 文件结构

```text
docs/
├── DEVELOPMENT_LOG.md
└── dev-notes/
    └── YYYY-MM-DD-任务标题.md
```

## 阶段一：方案确认后

立即创建任务文章，并在索引中标记为“进行中”。记录：

- 背景与目标；
- 当前问题；
- 技术栈、关键依赖版本和开发环境；
- 启动、测试和检查命令；
- 候选方案及最终选择；
- 为什么这样选择；
- 预计修改范围、风险和验证计划。

## 阶段二：实现完成后

补充：

- 实际修改文件及职责；
- 核心实现流程；
- 值得回看的关键代码；
- 遇到的困难、失败方案和根因；
- 最终解决方式；
- 与原计划的偏差。

## 阶段三：测试结束后

补充并整理为可发布文章：

- 测试项目、命令和结果；
- 未验证内容；
- 遗留问题与风险；
- 本次真正学到的知识；
- 后续可优化方向；
- 将索引状态更新为“完成”或“受阻”。

## 写作风格

- 主要写给几个月后的自己，不重复科普显而易见的基础概念。
- 重点记录决策原因、踩坑、关键代码和解决过程。
- 内容完整到稍作整理即可公开发布，但不为了篇幅写空泛总结。
- 不记录真实密钥、令牌、个人隐私或敏感生产数据。
